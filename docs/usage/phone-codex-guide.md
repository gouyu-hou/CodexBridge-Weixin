# 手机使用 Codex 详细文档

这份文档写给第一次使用 `CodexBridge Weixin Admin` 的用户。看完以后，你应该能做到：

- 用手机微信和电脑上的 Codex 对话。
- 用手机让 Codex 读取、修改、测试电脑里的项目代码。
- 在手机上切换会话、重命名会话、上传图片和文件。
- 在需要时审批 Codex 的文件修改或命令执行。
- 遇到无回复、卡住、API key 报错、端口冲突等问题时知道先检查哪里。

如果你只想日常聊天，可以只看前 4 章。如果你想用手机控制电脑上的项目代码，重点看第 5 章和第 6 章。

## 1. 这个功能是什么

`CodexBridge Weixin Admin` 的作用可以理解成一座桥：

```text
手机微信消息
  -> CodexBridge 本地服务
  -> 电脑上的 Codex / 模型接口
  -> 得到最终回答
  -> 发回微信
```

也就是说，你在手机微信里发消息，实际处理任务的是你电脑上正在运行的 CodexBridge 服务。它可以调用你配置好的 API key、模型、provider，也可以在你指定的项目目录里让 Codex 读代码、改文件、跑命令。

注意：手机不是直接连接模型。手机是通过你的电脑中转，所以电脑必须开机、联网，并且 CodexBridge 服务正在运行。

## 2. 使用前准备

### 2.1 电脑要求

推荐环境：

- Windows 10 或 Windows 11。
- 电脑能正常联网。
- 已安装或已解压运行 `CodexBridge Weixin Admin`。
- 已配置可用的 API key。
- 已登录或绑定微信 Bot / iLink Bot 入口。

如果是安装包用户，通常不需要自己安装 Node.js、npm、pnpm、tsx、Electron。安装包会把运行所需内容一起带上。

如果是开发者源码运行，才需要 Node.js 24 或以上版本，并在项目目录中执行 `npm install`。

### 2.2 手机要求

- 手机安装微信。
- 手机能扫描二维码。
- 你需要能和绑定后的 Bot 对话。

### 2.3 API key 和 provider

这个软件支持两种配置方式：

```text
方式一：手动填写 API key
方式二：跟随 CCSwitch / Codex 当前配置
```

手动填写适合不想装 CCSwitch 的用户。你在管理面板里填：

- provider 名称
- model 模型名
- API key
- Base URL

CCSwitch 模式适合已经使用 CCSwitch 管理多个 key 的用户。你在 CCSwitch 里切换账号或模型后，CodexBridge 可以同步使用当前配置。

如果你没有 API key，可以在管理面板里的链接进入 `ztoken.app` 获取兼容接口。文档里不会写入真实 API key，真实 key 只应该保存在你自己的本地配置里。

## 3. 启动和关闭

### 3.1 正常启动

如果你已经安装了桌面版：

1. 双击桌面上的 `CodexBridge Weixin Admin`。
2. 等待管理面板打开。
3. 顶部状态显示服务正常后，就可以在微信里使用。

如果你是源码开发方式：

```powershell
cd D:\IT_learn\codex_weixin\CodexBridge
npm run weixin:electron
```

### 3.2 正常关闭

如果你使用的是 Electron 桌面版，关闭软件窗口时，服务应该一起停止。

如果你发现窗口关了以后服务还在后台运行，可以重新打开管理面板查看服务状态，或者在任务管理器里结束相关的 `CodexBridge Weixin Admin` / `node.exe` 进程。

### 3.3 电脑关机或断网后会怎样

电脑关机、睡眠、断网、服务退出后，手机微信不能继续让本地 Codex 执行任务。因为真正运行 Codex 的地方是你的电脑。

恢复方式：

1. 打开电脑。
2. 重新启动 `CodexBridge Weixin Admin`。
3. 等待服务状态正常。
4. 在微信里发送 `/status` 或普通消息测试。

## 4. 第一次绑定微信

### 4.1 主账号绑定

主账号一般是你自己的微信入口。步骤：

1. 打开 `CodexBridge Weixin Admin`。
2. 进入管理面板的用户入口或二维码页面。
3. 生成登录二维码。
4. 用你的微信扫描二维码并确认。
5. 回到管理面板确认账号已在线。
6. 在微信里发送 `你好` 测试。

### 4.2 给朋友使用

如果你想让朋友也使用：

1. 在管理面板生成朋友入口二维码。
2. 把二维码图片或链接发给朋友。
3. 朋友用微信扫描并确认。
4. 管理面板里应该能看到新增用户。
5. 朋友发送消息后，消息会通过你电脑上的服务处理。

注意：

- 朋友的会话数据会保存在你的电脑数据目录中。
- 如果你的电脑关机，朋友也不能继续使用。
- 如果你的 API key 没额度或模型接口不可用，朋友也会受到影响。
- 建议给朋友设置合适权限，不要默认开放高风险项目操作。

### 4.3 二维码有效期

二维码通常不是永久有效的。如果状态变成 `expired`，说明这个二维码已经过期，需要重新生成一个新的二维码。

如果看到类似：

```text
status: wait
status: expired
```

意思是：一开始等待扫码，后来二维码过期。重新点击生成即可。

## 5. 手机普通聊天

绑定成功后，你可以像普通聊天一样发消息：

```text
帮我总结一下今天要做的事情
```

```text
帮我写一个朋友圈文案，语气自然一点
```

```text
解释一下什么是 API key，我是零基础
```

默认情况下，微信端应该只收到最终回答，不应该收到完整思考过程。如果你看到思考过程，说明过滤或模型输出格式可能有问题，可以先在管理面板检查配置，也可以用 `/status` 看当前状态。

## 6. 手机控制电脑上的项目代码

这是这个项目最重要的能力之一。你可以用手机告诉 Codex 去操作电脑上的某个项目目录。

### 6.1 基本原理

你先告诉 CodexBridge 项目在哪个目录，然后再发任务。

例如你的项目在：

```text
D:\IT_learn\codex_weixin\CodexBridge
```

你在微信里发送：

```text
/project D:\IT_learn\codex_weixin\CodexBridge
```

然后再发送：

```text
请读取这个项目，帮我检查为什么管理页面打不开，修复后运行测试，只把最终结果告诉我
```

这样 Codex 会在这个目录里工作。

### 6.2 `/project` 的三种常用用法

```text
/project
```

查看当前项目控制状态，包括当前项目目录、默认目录和权限状态。

```text
/project on
```

使用当前会话或默认目录开启项目控制。如果系统已经知道默认项目目录，就可以这样快速开启。

```text
/project D:\你的项目路径
```

指定一个明确项目目录。下一条普通任务消息会在这个目录里创建或绑定 Codex 项目会话。

```text
/project cancel
```

取消还没有开始的新项目控制会话。比如你刚发了 `/project D:\xxx`，但还没发任务，发现路径填错了，就可以取消。

### 6.3 推荐任务写法

好的任务要尽量包含 4 件事：

```text
目标 + 范围 + 验证方式 + 输出要求
```

示例：

```text
请读取这个项目，修复启动时报端口冲突的问题。只改必要文件，修复后运行相关测试，最后告诉我改了哪些文件和测试结果。
```

更明确的示例：

```text
请检查管理面板里“开始诊断”按钮为什么返回检查失败。先定位原因，再修复。不要改无关功能。修复后运行 admin_server 相关测试。
```

如果你只想看结果，可以加：

```text
只把最终结果发给我，不要输出思考过程。
```

### 6.4 权限和审批

手机控制项目时，Codex 可能会：

- 读取文件。
- 修改文件。
- 运行命令。
- 启动服务。
- 执行测试。

为了安全，某些操作会要求你在微信里审批。常见命令：

```text
/allow
```

查看当前等待审批的请求。

```text
/allow 1
```

批准第 1 个请求一次。

```text
/allow 2
```

批准并在当前会话中记住类似请求。

```text
/deny
```

拒绝当前审批请求。

如果你不确定某个命令是否安全，就不要批准。你可以让 Codex 先解释它要做什么：

```text
先解释一下你要执行的命令有什么作用，不要直接执行
```

### 6.5 权限模式

查看权限：

```text
/permissions
```

常见权限模式：

```text
/permissions default-permissions
```

推荐日常使用。允许在工作区内正常读写，越界或高风险操作会请求你审批。

```text
/permissions auto-review
```

工作区可写，部分审批由审查代理辅助判断。

```text
/permissions full-access
```

完全访问且尽量不审批。这个模式风险更高，只建议你明确知道任务需要它时使用。

建议新手先用默认权限，不要长期使用 `full-access`。

## 7. 图片和文件上传

### 7.1 单张图片

你可以直接给 Bot 发送图片，然后再发送说明：

```text
请看这张图，告诉我报错原因和解决步骤
```

### 7.2 多张图片一起分析

如果你想连续发多张图片，并让 Codex 一次性结合分析，推荐使用上传模式：

```text
/up
```

然后连续发送多张图片或文件。

最后发送你的提示词：

```text
请结合刚才所有截图，判断为什么服务启动失败，并给我按步骤排查
```

### 7.3 上传模式命令

```text
/up
```

开启上传暂存模式。开启后，图片和文件会先暂存，不会每张都立刻触发回复。

```text
/up status
```

查看当前已经暂存了哪些文件。

```text
/up cancel
```

取消上传模式，并清空这次暂存内容。

### 7.4 上传模式注意事项

- 开启 `/up` 后，不需要每发一张图片都写说明。
- 发送最终文字提示词后，才会统一提交给 Codex。
- 如果你不想继续这次上传，记得发 `/up cancel`。
- 如果图片太多或太大，处理速度会变慢。
- 电脑端服务必须保持运行，否则上传内容无法继续处理。

## 8. 会话管理

CodexBridge 支持多个历史会话。你可以在手机微信上创建、查找、切换和重命名会话。

### 8.1 新会话

```text
/new
```

准备一个新会话。

注意：为了避免产生很多空会话，通常是你发送 `/new` 后，再发送第一条普通内容，系统才真正创建新会话。

示例：

```text
/new
```

```text
这个会话专门用来学习 Python，请以后围绕 Python 给我讲解
```

### 8.2 查看历史会话

```text
/threads
```

查看历史会话列表。

列表里当前会话前面会有醒目标识。每一项通常会包含：

- 序号。
- 会话标题。
- 最近一次问题或预览。
- 更新时间。

### 8.3 翻页

```text
/next
```

查看下一页历史会话。

```text
/prev
```

查看上一页历史会话。

你需要先发送 `/threads` 或 `/search`，再使用 `/next`、`/prev`。

### 8.4 搜索会话

```text
/search 手机买球分析
```

搜索标题或内容里包含相关关键词的会话。

### 8.5 打开会话

按序号打开：

```text
/open 2
```

按名字打开：

```text
/open 手机买球分析
```

如果有多个同名或相似名字，建议先 `/search 名字`，确认序号后再 `/open 2`。

### 8.6 预览会话

```text
/peek 2
```

预览列表里第 2 个会话的最近内容，但不切换过去。

### 8.7 重命名会话

给当前会话改名：

```text
/rename this 手机买球分析
```

给列表里的第 2 个会话改名：

```text
/rename 2 项目学习
```

如果你已经知道名字，也可以用搜索再打开：

```text
/search 项目学习
/open 项目学习
```

建议会话名字取得具体一点，比如：

```text
接入微信聊天
项目代码解答
手机买球分析
AI 大模型应用学习
```

不要都叫 `新线程00001`，后面很难找。

### 8.8 归档、置顶和查看全部

归档列表第 2 个会话：

```text
/threads del 2
```

置顶列表第 2 个会话：

```text
/threads pin 2
```

取消置顶：

```text
/threads unpin 2
```

查看全部会话，包括归档项：

```text
/threads all
```

注意：这里的删除很多时候更接近“从当前列表归档/隐藏”，不一定等于彻底删除底层 Codex 历史文件。

## 9. 模型和 provider

### 9.1 查看当前 provider

```text
/provider
```

它会显示当前使用哪个 provider 配置。

### 9.2 切换 provider

示例：

```text
/provider openai-default
```

```text
/provider DeepSeek
```

具体可用 provider 取决于你管理面板里的配置。

### 9.3 查看当前模型

```text
/model
```

会显示当前模型、模型来源、推理强度等信息。

### 9.4 查看可用模型

```text
/models
```

列出当前 provider 下可选模型。

### 9.5 切换模型

按模型名切换：

```text
/model gpt-5.5
```

按序号切换：

```text
/models
/model 1
```

切换模型并设置推理强度：

```text
/model gpt-5.5 high
```

```text
/model gpt-5.5 xhigh
```

恢复默认：

```text
/model default
```

不同模型支持的推理强度不一定一样。如果模型不支持某个强度，系统会按能力做兼容或提示。

## 11. 常见使用场景

### 11.1 让 Codex 修项目 bug

```text
/project D:\IT_learn\codex_weixin\CodexBridge
```

```text
请检查为什么双击启动后管理页面打不开。先定位原因，再修复。修复后运行相关测试，最后只告诉我改动文件和测试结果。
```


```text
/up
```

然后连续发截图。

最后发：

```text
请结合这些截图判断问题原因，给我一步一步解决方法。不要输出思考过程，只给最终结论。
```

### 11.3 新建一个专门学习的会话

```text
/new

```

```text

```

```text
/search 源码学习
```

```text
/open CodexBridge源码学习
```

如果同名或没找到：


## 12. 常见问题和解决方法

### 12.1 微信上一直显示“对方正在输入”

可能原因：

- 当前任务还在运行。
- 模型接口很慢。
- 网络不稳定。
- Codex 卡在审批或命令执行。
- 微信侧状态没有及时结束。

处理顺序：

```text
/status
```

看当前状态。

```text
/stop
```

停止当前任务。

如果还不行：

```text
/reconnect
```

最后可以在管理面板里重启服务。

### 12.2 发两条消息提示“有一轮回复在进行中”

同一个会话通常是单会话排队，避免多个任务同时改同一批文件造成混乱。

处理方式：

- 等当前回复完成。
- 或先 `/stop` 停止当前回复。
- 如果是想补充提示，可以直接发一条追加说明，系统支持在思考时追加提示的版本会把它合并进当前任务或下一轮处理。

### 12.3 502 / 503 是什么

常见含义：

- `502 Bad Gateway`：上游模型服务临时不可用，通常不是本项目代码错误。
- `503 Service Unavailable`：模型服务临时不可用或连接失败。

处理方式：

1. 等几分钟重试。
2. `/retry` 重试上一条。
3. `/reconnect` 刷新连接。
4. 检查 API key、额度、Base URL。
5. 管理面板里运行一键诊断。

### 12.4 429 是什么

`429 Too Many Requests` 通常表示：

- API key 没额度。
- 请求太频繁。
- 当前套餐限速。
- provider 限制并发。

处理方式：

- 更换有额度的 API key。
- 降低并发。
- 等待额度恢复。
- 在 CCSwitch 中切换可用账号，然后同步配置。

### 12.5 朋友扫码显示暂时无法连接

检查：

- 你的电脑是否开机并联网。
- `CodexBridge Weixin Admin` 是否正在运行。
- 管理面板里微信账号是否在线。
- 二维码是否过期。
- 朋友是否已经确认授权。
- API key 是否可用。

可以在管理面板点“一键诊断”，或在微信里发：

```text
/status
```

### 12.6 API key 换了以后还是报错

如果你是手动填写：

1. 打开管理面板。
2. 进入模型/provider 配置。
3. 填入新的 API key。
4. 保存。
5. 重启或 `/reconnect`。

如果你用 CCSwitch：

1. 在 CCSwitch 里切换到新的 key。
2. 回到管理面板点同步 CCSwitch。
3. 或等待自动同步。
4. 微信里发 `/reconnect`。

### 12.7 为什么手机不能看到电脑 Codex 侧边栏

微信会话和 Codex 桌面侧边栏不是完全同一个界面。CodexBridge 会尽量把微信创建和命名的会话同步到本地索引里，但 Codex 官方桌面侧边栏是否立刻显示，取决于底层 Codex 历史、线程文件和当前版本支持。

你在微信里管理会话，优先使用：

```text
/threads
/search 名字
/open 名字
/rename this 名字
```

### 12.8 为什么电脑一断网微信就不能用了

因为 CodexBridge 是本地桥接服务，模型请求、会话管理、项目操作都依赖你的电脑。如果电脑断网，手机发来的消息无法正常转给模型或 Codex。

想稳定使用：

- 保持电脑开机。
- 保持网络连接。
- 不要让电脑自动睡眠。
- 保持 `CodexBridge Weixin Admin` 运行。

### 12.9 怎么避免 C 盘占用越来越多

建议：

- 把项目目录放到 D 盘，例如 `D:\IT_learn\codex_weixin\CodexBridge`。
- 把数据目录放到 D 盘，例如 `D:\IT_learn\codex_weixin\CodexBridgeData`。
- 定期在管理面板清理日志。
- 不要把 `release`、`node_modules`、`BuildTemp` 这类构建目录长期保留多个副本。

注意不要手动删除正在使用的数据目录，否则会丢失会话、账号绑定或配置。

## 13. 安全建议

手机控制电脑项目很方便，但也要注意安全：

- 不要把 API key 发给别人。
- 不要把 `weixin.service.env` 上传到 GitHub。
- 不要随便给朋友 `full-access` 权限。
- 不确定的审批请求先拒绝。
- 重要项目改动前先提交 Git 或备份。
- 让 Codex 执行删除、移动、覆盖文件之前，先让它说明影响范围。
- 不要让陌生人扫码使用你的 Bot。

## 14. 推荐新手流程

如果你是第一次用，按这个顺序来：

1. 打开 `CodexBridge Weixin Admin`。
2. 在管理面板配置 API key / provider / model。
3. 生成微信二维码并扫码绑定。
4. 微信里发送 `你好` 测试普通聊天。
5. 微信里发送 `/status` 看状态。
6. 微信里发送 `/new` 创建第一个正式会话。
7. 微信里发送 `/rename this 我的第一个会话` 给会话取名。
8. 如果要控制项目，发送 `/project D:\你的项目路径`。
9. 发送明确任务，让 Codex 操作项目。
10. 有审批时用 `/allow 1`，不确定就 `/deny`。
11. 做完后用 `/threads` 和 `/search` 管理历史会话。
